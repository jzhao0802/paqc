paqc.driver package¶

Submodules¶

paqc.driver.driver module¶

This submodule contains the main class that takes in the config file and executes the QC scripts in the requested order and collates their findings in a report.

class paqc.driver.driver.Driver(config_path, verbose=True, debug=False, to_hash=False, df_input=None)¶

Bases: object

Main executor class of the the QC pipeline. It reads in a YAML config file, checks it, then executes the tests on the specified input files one by one, while collecting their output and generating a report from it.

config_loader()¶

This will load, check and parse the config file for the driver.

Returns:Nothing. Updates the internal config object of the driver.
data_loader(input_file_path)¶

Loads an input data file using its path and the source argument of the config file.

Parameters:input_file_path – path to the data.
Returns:pandas DataFrame object of the fully loaded datafile.
do_compare_qc(input_file1, input_file2, qc)¶

Function for executing QCs on two input dataframes at once. If the dataframes are already loaded, we use them, otherwise we load them too.

Parameters:
  • input_file1 – input1,…,input_n in general part of config
  • input_file2 – input1,…,input_n in general part of config
  • qc – string, name of qc to execute on the two dataframes.
Returns:

Nothing, updates Driver’s internal report object.
do_qc(input_file, input_file_path, qcs)¶

This is the function that actually takes an input data file with a path, loads it, then calls all required qc functions on it.

Parameters:
  • input_file – input1,…,input_n in general part of config
  • input_file_path – actual file path to the data
  • qcs – dictionary of qcs to execute on a given data file.
Returns:

Nothing, updates Driver’s internal report object.
main()¶

This is the main function or the ‘brain’ of the Driver class, which will execute the qc functions one by one of the desired input files even if those have multiple chunked data files within them. If we encounter QCs that are comparing two dataframes, we load both required files and perform the qc.

Returns:Nothing, calls the do_qc() on each data_file and executes the required qc functions, then generates the .csv and HTML report
printer(to_print, hline_before=False, hline_after=False)¶

Simple wrapper function, that prints messages to users if the driver object was instantiated with verbose=True.

Parameters:
  • to_print – Message to print.
  • hline_before – Boolean, should a horizontal like be printed before the to_print string?
  • hline_after – Boolean, should a horizontal like be printed after the to_print string?
Returns:

Nothing, prints to standard out.
run(generate_report=True)¶

Runs the QC pipeline by executing the various methods of the Driver class in succession.

Parameters:generate_report – Boolean, if set to False Driver will not generate report, but simply run the QCs.
Returns:None
